因為它不是原始著述，不需要有自己的 canonical transcription。它是由其他資料生成的。

所以它的性質不是：

derived work

    ├── 年譜匯總
    └── …

這個抽象很有用。

2. 校記與異文可以統一

我很同意你的判斷：

校記 is no different than 異文.

更準確地說：

異文是 variant data；校記是 variant commentary。

它們不是兩種完全不同的東西，而是同一個現象的兩個層面。

例如：

異文：
A 本作「風急」

校記：
某本作「風急」，按上下文當從某本。

兩者都圍繞同一個核心：

某一個 textual locus 上存在 variant evidence。

因此可以統一為：

Variant Node
├── variant readings
├── sources / editions
├── notes / comments
├── editor judgment
└── references

這樣一來，校記不必獨立成一種全新機制。

它可以是 variant apparatus 的一部分。

3. 但「校記作為杜著述」也仍然成立

你說：

If 校記 is to be treated as another 杜著述...

我覺得這裡要分兩層。

第一層：

校記作為資料類型，屬於 variant apparatus。

第二層：

某一部書中的校記，可以作為一個可抽取、可建樹、可匯總的 textual layer。

所以它可以同時是：

data type = 校記
source work = 蕭校記 / 仇校記 / 楊校記 / 謝校記
aggregate work = 校記匯總

這不矛盾。

就像「注釋」既是一種 metadata 類型，也可以作為某部杜著述中的一個 textual layer。

4. 年譜更像是 timeline tree

年譜這個想法很有意思，因為它不是以詩碼為主軸，而是以時間為主軸。

也就是：

杜詩系統常規樹：
詩 → 行 → 句 → 字

年譜樹：
年 → 事件 → 證據 / 詩 / 著述資料

你說約 60 個年份節點，我覺得很自然：

年譜
├── 712
│   ├── 蕭滌非/張忠綱資料
│   ├── 其他年譜資料
│   └── 相關詩文
├── 713
├── …
└── 770

這其實又是一種 derived 杜著述。

它不是從一部書的線性文本出發，而是從一個 organizing principle 出發：

organize Du Fu data by year.

這和資料匯總很像，只是 資料匯總 的 organizing principle 是詩/坐標，年譜 的 organizing principle 是年份。

5. 我會暫時這樣命名這個新概念

先不要提升成大原則，但可以作為小部件定義：

Aggregate Work（匯總著述）

一種由其他杜著述或資料來源生成的著述型資料集合。它本身未必有 canonical_text，也未必有 metadata 原始資料；它主要保存由多個來源重新組織而成的 trees 與 views。

它的例子包括：

資料匯總
校記匯總
年譜匯總
詞典匯總

我覺得這個概念現在已經足夠具體，可以先寫進 documentation 的小定義裡，不必上升到整個 framework 的 design principle。
