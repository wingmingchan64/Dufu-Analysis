Canonical Annotation Rules (10 Core Rules)

1️⃣ Canonical Text

所有 annotation 均以 canonical 杜甫文本 為基準。
canonical 文本的字序固定，不因版本差異改動。

2️⃣ Coordinate System

文本坐標採四層結構：

詩 : 聯 . 句 . 字

字序為 句內字序。

3️⃣ Anchor

每條 annotation 必須有 anchor。
anchor 必為 單一字坐標。

4️⃣ Scope

每條 annotation 必須有 scope。
scope 為 canonical 文本上的 連續字序區間。

5️⃣ Containment Rule

anchor 必須包含於 scope。

anchor ∈ scope

6️⃣ Scope Levels

scope 可為：

字
詞
句
聯
詩

若無法確定細範圍，可提升至較高層級。

7️⃣ No Null Scope

annotation 不允許 scope = null。
若範圍不明，可使用整首詩作為 scope。

8️⃣ Interval Rule

annotation scope 只允許：

相離（disjoint）

包含（nested）

不允許：

交叉（crossing）

9️⃣ Controlled Annotation Types

annotation type 使用 固定分類，例如：

字注
詞注
句注
聯注
詩評
泛論

🔟 Single Primary Classification

annotation 原則上只屬於 一個主要類型與主題。
若內容複雜，優先 分解 annotation，而非多重分類。

五、這 10 條其實就是你整個系統的「憲法」

之後不管你寫：

JSON schema

PHP / Python 程式

metadata 編輯器

詞典生成器

都會回到這 10 條。
