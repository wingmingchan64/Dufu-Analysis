## 槪述

「默認版本定位」部分要展示的，是這個系統在爲杜甫詩文定位方面，能幹甚麼，能達到的效果。至於如何達到這種效果，則是「坐標、路徑」部分要展示的。

- 杜甫詩文的默認版本內容儲存於基準正文樹中
- 杜甫詩文的任何片段，必定儲存於某樹的某些節點中
- 要到達這些節點，必須依循某獨特的路徑（可用坐標來表示）
- 杜甫詩文的任何片段必定與某獨特路徑相對應
- 獨特路徑可被視爲與之相對應的杜甫詩文片段的主鍵（primary key）
- 杜甫詩文的默認版本是個封閉的系統，因此路徑→詩文對照表也是有限的（finite）

## 如何定位

- 系統的定位機制主要以詩句爲最大單位
- 句內定位是要決定某字在包含該字的詩句的句內位置
- 大於句的單位，如行、段、首，則以句內定位爲支撐點
- 杜詩最短的詩句有兩個字，最長的詩句有十一個字
- 系統以基準正文樹爲基礎，生成十一個 N 字組合→坐標、坐標→N 字組合對照表（系統內部自動作坐標、路徑轉換）
- 系統也生成行→坐標之類的對照表

以<a href="https://github.com/wingmingchan64/Dufu-Analysis/blob/main/schemas/json/coords/一字組合_坐標.json">一字組合_坐標.json</a>的「岱」字爲例：

<pre>
    "岱": [
        "〚0003:3.1.1〛",
        "〚0008:4.1.5〛",
        "〚1252:7.1.7〛"
    ],
</pre>

在所有的杜詩中，「岱」字出現了三次，分別是0003《望嶽》、0008《登兗州城樓》、1252《洗兵馬》。一個像"〚0003:3.1.1〛"的坐標，系統會自動轉換成路徑"0003,3,1,1"。有了這個坐標/路徑，系統就知道「岱」字的實際位置。

要知道 X 跟 Y 兩個字是否同時出現在同一首詩之中，系統只要比較這兩個字的坐標/路徑集，看看兩個集裏，有沒有共同的詩碼。要知道 X 跟 Y 兩個字是否同時出現在同一首詩、同一行中，則要同時看二字共同的詩碼、行碼。例如「賦料揚雄敵。詩看子建親」中的「料」、「看」二字，是否還出現在其他詩的同一行中，系統會比較以下兩個坐標/路徑集：

路徑_句.json 則是一個路徑→句的對照表：

<pre>
    "0003,3,1": "岱宗夫如何",
    "0003,3,2": "齊魯青未了",
    "0003,4,1": "造化鍾神秀",
    "0003,4,2": "陰陽割昏曉",
</pre>

系統內有數十個這種生成的對照表，系統就是利用這些對照表爲杜詩詩文定位的。
