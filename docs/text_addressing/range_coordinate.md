# Range Coordinate （By ChatGPT）

Status: Draft

## 槪述

本文定義「範圍路徑（range coordinate）」的語法與語義，以及其與一般坐標（coordinate）與 scope（路徑集）之間的關係。

本文件的核心原則是：

>範圍坐標不是本體表示，而是對特定條件下之路徑集（path set）的壓縮表示。

---

## 基本槪念

### Path

Path 是基準正文樹中的一條完整路徑，例如：

〚LUNYU,01,1,3,2,4〛

表示從 root 到某一葉節點（通常為「字」）的唯一位置。

2.2 Scope（路徑集）

Scope 定義為：

一組有序的、合法的 path。

例如：

{〚…,1〛, 〚…,2〛, 〚…,3〛}

可對應：

字範圍
句範圍
段範圍
行集合
搜索結果
2.3 範圍坐標（Range Coordinate）

範圍坐標是對某些特殊 scope 的壓縮表示，例如：

〚…,1-3〛

其語義為：

〚…,1-3〛 ≡ {〚…,1〛, 〚…,2〛, 〚…,3〛}
3. 語法

範圍僅允許出現在最後一層：

〚doc_id, n1, n2, ..., nk〛
〚doc_id, n1, n2, ..., nk-m〛

其中：

nk-m 表示範圍
nk ≤ m
4. 語義條件（Range Representability）

一個 scope 可以表示為範圍坐標，當且僅當滿足以下條件：

4.1 同一父節點

所有 path 必須：

共享相同的 prefix（除最後一層外完全一致）
4.2 最後一層差異

所有 path 僅在最後一層不同。

4.3 有序且連續（無空缺）

最後一層索引必須構成：

n, n+1, n+2, ..., m

不得出現：

缺失（如 1,2,4）
重複
無序
4.4 範例
合法
{1,2,3,4,5} → 〚…,1-5〛
非法（有空缺）
{1,2,4,5} → 不可表示為單一範圍坐標
5. 關於「空缺」
5.1 定義

若某一層 index 在樹中不存在，則該位置視為空缺。

例如：

行碼中存在空行分隔段落
導致某些行碼在結構上不存在
5.2 結論

只要存在空缺，該 scope 即不可表示為單一範圍坐標。

這不表示該 scope 不合法，而僅表示：

該 scope 無法壓縮表示。

6. 行（line）之特殊情況

在某些文本（如《詩經》）中：

空行用於分隔段落或章節
導致行碼不形成稠密序列

例如：

行碼：5, 7（6 不存在）

則：

〚…,5-7〛 → 非法範圍坐標

但以下仍然是合法 scope：

{〚…,5〛, 〚…,7〛}
7. 本體與表示的區分

本系統區分：

7.1 本體（Scope）
路徑集（path set）
唯一語義載體
7.2 表示（Coordinate / Range Coordinate）
單一路徑表示
或壓縮表示（range）
7.3 原則

所有範圍坐標皆可展開為 path set；
並非所有 path set 均可壓縮為範圍坐標。

8. 函式語義（建議）
8.1 範圍驗證函式
是合法範圍路徑( $input, &$ary, $options )

語義：

檢查輸入是否構成可壓縮之範圍路徑
回傳 bool
8.2 輸出參數 $ary
在函式開始時初始化為空陣列
僅在回傳 true 時填入內容：
[
    '〚…〛' => '文字',
    ...
]
在回傳 false 時保持為空陣列
8.3 $options

用於控制文字提取方式（例如是否加句號），
不影響範圍合法性判斷。

9. 設計原則
9.1 嚴格性
範圍坐標必須滿足「無空缺」
不允許模糊或推測性補全
9.2 分離性
坐標語言不承擔搜索功能
不表達非連續集合
9.3 可逆性

對任一合法範圍坐標：

coordinate → path set → text

必須穩定且可預測

10. 不支持的情形

本系統刻意不支持：

非連續集合範圍（如 {1,3,5}）
多段範圍（如 1-3,5-7）
跨層範圍
自動忽略空缺的範圍
11. 小結

本設計採取以下立場：

範圍坐標是一種受限且嚴格的壓縮表示，僅適用於無空缺的結構性連續路徑集。

當該條件不滿足時：

應回退至顯式路徑集表示，而非放寬範圍語義。
