# 基準正文樹

基準正文樹是以某文檔中的基準正文，通過程式處理而生成的、儲存基準正文的所有內容的樹形結構。這樹形結構以 JSON 的陣列形式儲存於某 `.json` 文檔中；利用這個數據結構，用者可以根據需要，在樹中刪除、替換或插入文字、其他樹形結構，以生成各種版本、注本、評本。

這樹形結構，基本反映基準正文的文字結構（語義結構），如以段、闕、行、句、字等語言單位爲樹的節點，終端節點則是基準正文中的每一個字。

基準正文樹的操作，一般以後設資料標記爲操作指令。

## 基準正文與基準正文樹的區別

基準正文是某詩、某文的基準文本，是某古籍中的一段文字。與此相應的基準正文樹，則是儲存該段文字的樹型數據結構。

從處理程式的角度看，基準正文只是一段文字；文字操作模式只限於全文搜索、整體的文字插入、替換，頂多再加上正規表達式（regular expressions）。這種操作模式有嚴重的局限性，不能百分之百、準確地生成目標文檔。以甲戌本《紅樓夢》爲例，正文中有「箇」、「個」，詞義基本一樣，可以互換。基準正文則只能用其中一個，另一個得用轉換規則生成。這個轉換沒有規則可循，因此是不可能的。

要注意的是，處理文字時，不能用第幾句、第幾個字的概念。但凡用上這些概念，就已經離開文字的範疇，進入數據結構的範疇。單是用簡單的線性排序，比如說，從第一個字開始數，數到第幾個字，就等同於把一段文字轉化成一個一維陣列（one-dimensional array），那已經不是原來的那段文字了。一進入到數據結構的範疇，一維陣列就遠遠比不上樹形結構，因爲樹形結構支持各種强大的樹形操作功能，而一維陣列只支持線性搜索。

## 基準正文樹爲甚麼存在

- 基準正文樹只儲存某詩、某文的文字內容
- 作爲數據結構，基準正文樹容許外挂資料，也支持各種樹操作
- 基準正文樹支持各種文本的格式（一般文字文本、xml、md、html 等等）
- 以基準正文樹爲基礎，結合由此而衍生的坐標系統，可以生成各種基準正文-坐標索引，支持各種各樣的搜索操作
- 注本、評本幷不獨立存在於系統中
- 注文、評論、異文等，以外挂的機制、後設資料的形式，通過程式挂到基準正文樹上
- 同一棵樹、同一組後設資料可以以不同的面貌（views）出現
- 由於基準正文樹與後設資料的完全分離，同一棵樹可以同時外挂幾組不同的後設資料（卽不同注本、不同版本的幷列）
- 由於後設資料完全獨立於基準正文樹之外，後設資料本身可支持各種索引、標簽、統計、分類等操作
