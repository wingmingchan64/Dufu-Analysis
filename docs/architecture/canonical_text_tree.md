# 基準正文樹 （Canonical Text Tree）

`基準正文樹`是詩文的一種結構化表示方式。`詩文文本`被表示爲一棵層級結構的樹（如：詩 → 行 → 句 → 字），其中「字」爲最小單位。該結構可作爲文本處理、各種版本的注本、評本的生成基礎。

## 優點

### 1．單一文本來源，多版本生成

一棵基準正文樹可以生成多種不同的文本版本，例如：

- 異文版本
- 帶夾注版本
- 不同校本
- 不同排印形式

因此只需維護一個基準文本，卽可派生多種版本。

### 2．支持靈活的結構操作

由於文本以樹結構表示，可以進行多種結構性操作，包括：

- 字級操作：刪除、替換或插入單個字
- 子樹操作：插入或刪除整句、整行或整聯
- 結構調整：互換兩個非終節點（如兩句或兩行的位置）

這些操作可用於處理異文、補字、版本差異等情況。

## 基準正文樹來源

基準正文樹由 canonical .txt 文本經程序生成。 .txt 爲唯一人工編輯的文本來源，其餘資料（坐標表、正文樹、JSON 等）均由程序派生生成。

	canonical .txt
			↓
	coordinate system
			↓
	canonical text tree

## 基準正文樹 vs. TEI

本系統不直接使用 TEI XML 作爲 canonical representation，而採用結構化正文樹。 TEI、HTML 或其他標記語言可以由正文樹生成，而不作爲原始文本。

## 版本生成

	canonical tree
			↓
	edition generator
			↓
	many editions

## 樹例

### 結構圖

	Poem
	 ├─ Line 3
	 │   ├─ Segment 1
	 │   │   ├─ Char 1
	 │   │   ├─ Char 2
	 │   │   └─ ...
	 │   └─ Segment 2
	 │       ├─ Char 1
	 │       └─ ...
	 └─ Line 4

### Controller

基準正文樹只是一個 `model`的起點，一個只有杜甫詩文的結構。任何的編輯、改動，都得依賴 `controller` 與後設資料 （metadata）指令。後設資料好比挂在樹上的裝飾品，而 controller 則負責處理後設資料、結合基準正文樹與後設資料而生成各種各樣的面貌（`views`）。

### 其他用例

- 加入 markup、 HTML tags 等等
- 加入 emojis、 media links
- 加入外語譯文
- 加入現代標點符號（如奇句用逗號，偶句用句號）
- 更換整首詩的文字內容

### 可能的系統延伸

- 文字評論、比較
- 異文比較
- 不同層次的注釋
- 文本分析、比較
- 版本、次版本的生成
- 後設資料的分類、索引、樹結構化
- 支持多種編程語言
- 面貌模板（view templates）

### 基準正文樹的普遍性

經實驗證明，基準正文樹模式不僅適用於杜詩，也適用於唐詩、宋詞及一般古籍文本；其生成過程簡單穩定，困難主要不在於樹的生成，而在於基準文本的整理與後續應用層的設計。
