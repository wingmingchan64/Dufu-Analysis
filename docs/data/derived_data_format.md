# 衍生資料格式（Derived Data Formats）

從 canonical `.txt` 文本可自動生成若干衍生資料，用於不同的處理與應用場景。目前主要生成兩類結構化資料：坐標表（coordinate map）與基準正文樹（canonical text tree）。每一類資料均提供兩種形態：整體（corpus-level）與單篇（document-level）。

## 1．坐標表（Coordinate Map）

坐標表以「坐標 → 詩文」爲基本形式，用於快速檢索與索引建立。

例：

{
	"〚0003:3.1.1〛": "岱",
	"〚0003:3.1.2〛": "宗",
	"〚0003:3.1.3〛": "夫",
	...
}
### 1.1 整體坐標表

整個 corpus 合併爲一個檔案，通常以文檔碼爲索引鍵，例如：

	{
		"0003": {
			"〚0003:3.1.1〛": "岱",
			"〚0003:3.1.2〛": "宗",
			"〚0003:3.1.3〛": "夫",
			...
		},
		"0008: {
			...
		},
		...
	}

用途：

- 全庫文本搜索
- 建立字或句的索引
- 批量分析與統計

## 1.2 單篇坐標表

每一篇詩文生成一個獨立檔案，例如：

	447-001.json
	447-002.json

用途：

- 單篇處理與調試
- 局部檢索
- 減少載入整個 corpus 的開銷

## 2．基準正文樹（Canonical Text Tree）

基準正文樹是文本的結構化表示。文本被組織爲層級結構：

	詩 → 行 → 句 → 字

例：

	{
	  "3": {
		"1": {
		  "1": "亂",
		  "2": "雪",
		  "3": "千"
		},
		"2": {
		  "1": "新",
		  "2": "絲"
		}
	  }
	}

用途：

- 生成不同文本版本
- 插入夾注或校記
- 處理異文
- 進行結構性操作（如插入、刪除或移動句段）

2.1 整體正文樹

整個 corpus 合併爲一個大型 JSON 結構，例如：

{
  "447-001": { ... },
  "447-002": { ... }
}

用途：

- 全庫文本生成
- 批量版本處理
- 整體結構分析

### 2.2 單篇正文樹

每一篇詩文生成一棵獨立的正文樹，例如：

	447-001.json
	447-002.json

用途：

- 單篇文本生成
- 局部修改與測試
- 人工檢視與調試

## 3．設計原則

所有衍生資料均由 canonical .txt 文本自動生成。 `.txt ` 文件始終是唯一的文本來源，而 JSON 結構僅作爲程式處理與分析使用的中間資料。

這一設計使得：

- 文本保持簡潔與可讀
- 結構化資料可以自由生成或重建
- 文本內容與計算結構彼此分離
