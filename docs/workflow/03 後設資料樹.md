# 03 後設資料樹

Status: Draft

---

## 槪述

後設資料的是連結杜甫詩文的基準正文樹、杜著述/古籍正文樹二者的橋梁。基準正文樹只儲存杜甫詩文，而杜著述/古籍的正文樹則只儲存基本不含杜甫詩文的文字內容。要生成一個面貌，回復某杜著述的本來樣貌，或生成其他混合版本，必須把二者有系統地連結起來。

後設資料的儲存形式，同樣是樹結構。每條從樹根到終端節點的完整路徑，代表的就是一個具體的指令，指示系統如何把杜著述/古籍中的文字，插入到基準正文樹中適當的地方（如注釋、夾注），或者用這些文字來替換基準正文樹中的文字（如異文）。

---

## 後設資料樹的結構

先看一棵後設資料樹的例子：

<pre>
{
    "JINGQUAN": {
        "0002": {
            "題解": {
                "0003,詩題,a": {
                    "JINGQUAN,0002,1": "insert"
                }
            },
            "夾注": {
                "0003,5,1,4": {
                    "JINGQUAN,0002,6": "insert"
                },
                "0003,5,2,2": {
                    "JINGQUAN,0002,8": "insert"
                }
            },
            "旁注": {
                "0003,4,2,3": {
                    "JINGQUAN,0002,4": "insert"
                }
            },
            "注釋": {
                "0003,3,2,1-5": {
                    "JINGQUAN,0002,2": "insert"
                },
                "0003,4,1,1-5": {
                    "JINGQUAN,0002,3": "insert"
                },
                "0003,4,2,1-5": {
                    "JINGQUAN,0002,5": "insert"
                },
                "0003,5,1,1-5": {
                    "JINGQUAN,0002,7": "insert"
                },
                "0003,5,2,1-5": {
                    "JINGQUAN,0002,9": "insert"
                }
            },
            "眉批": {
                "0003,a": {
                    "JINGQUAN,0002,10": "insert"
                }
            }
        }
    }
}
</pre>

- 這是楊倫《杜詩鏡詮》 0002 《望嶽》的後設資料樹
- 樹根 `JINGQUAN` 是著述碼，指《杜詩鏡詮》
- `0002` 是《杜詩鏡詮》的詩碼，指《望嶽》
- `0002` 之下有 `題解`、`夾注`、`旁注`、`注釋`、`眉批` 五個子節點
- `題解` 的子節點是 `0003,詩題,a`
- `0003` 是默認版本（系統專用）《望嶽》的詩碼
- `0003,詩題,a` 是基準正文樹中的一條路徑，指向一個跟詩題有關的錨
- `0003,詩題,a` 的子節點是 `JINGQUAN,0002,1`
- `JINGQUAN,0002,1` 指向的是《杜詩鏡詮》的一段關於《望嶽》的文字：「《元和郡縣志》：泰山一曰岱宗，在兗州乾封縣西北三十里。」
- `insert` 表示此段文字應該插入到 `0003,詩題,a` 最後的位置
- `insert` 保留該位置中原有的文字（appending）

生成這樹的代碼：

<pre>
$默文檔碼 = '0003';
$著述碼   = 'JINGQUAN';
$版文檔碼 = '0002';
$m_tree = 生成後設資料樹( $默文檔碼, $著述碼, $版文檔碼 );
</pre>

## 後設資料樹的生成

- 任何一條後設資料樹中的完整路徑，旣是一個指令，但同時，至少是路徑的前半部，也是一個文件的所在路徑
- 以下是名爲「杜詩鏡詮」的文件夾內部結構：
<pre>
	- 杜詩鏡詮
	  - canonical_text
		- 0001.txt
		- 0002.txt
		  ...
	  - coordinates
	  - metadata
	   - 0001
	   - 0002
	     - m_tree.json
		 - 夾注.txt
		 - 旁注.txt
		 - 注釋.txt
		 - 眉批.txt
		 - 題解.txt
	   - 0003
		 ...
	   ...
</pre>
- `m_tree.json` 是用程式生成的，儲存的就是 0002 的後設資料樹
- 這樹的資料來自其他幾個 `.txt` 文檔
- `.txt` 文檔是手編的，內容是後設資料指令的後半部
- `0002/旁注.txt` 的內容：
<pre>
	{"scope":"0003,4,2,3","src_path":"JINGQUAN,0002,4"}
</pre>
- 爲了方便閱讀，"scope" 的値可以是詩文：
<pre>
	{"scope":"割","src_path":"JINGQUAN,0002,4"}
</pre>
- 以詩文作爲 "scope" 的値，此詩文在此詩中只能出現一次；就是說，詩文必須能轉換爲唯一的一條基準正文樹路徑

## 後設資料指令集

把這後設資料樹攤平，可以得到以下的指令集：

<pre>
Array
(
    [0] => JINGQUAN_0002_題解_0003,詩題,a_JINGQUAN,0002,1_insert
    [1] => JINGQUAN_0002_夾注_0003,5,1,4_JINGQUAN,0002,6_insert
    [2] => JINGQUAN_0002_夾注_0003,5,2,2_JINGQUAN,0002,8_insert
    [3] => JINGQUAN_0002_旁注_0003,4,2,3_JINGQUAN,0002,4_insert
    [4] => JINGQUAN_0002_注釋_0003,3,2,1-5_JINGQUAN,0002,2_insert
    [5] => JINGQUAN_0002_注釋_0003,4,1,1-5_JINGQUAN,0002,3_insert
    [6] => JINGQUAN_0002_注釋_0003,4,2,1-5_JINGQUAN,0002,5_insert
    [7] => JINGQUAN_0002_注釋_0003,5,1,1-5_JINGQUAN,0002,7_insert
    [8] => JINGQUAN_0002_注釋_0003,5,2,1-5_JINGQUAN,0002,9_insert
    [9] => JINGQUAN_0002_眉批_0003,a_JINGQUAN,0002,10_insert
)
</pre>

這是代碼：

<pre>
$paths = array();
record_mtree_paths( $m_tree, '' );
</pre>

也可以單提取 `注釋` 部分：

<pre>
record_mtree_paths( $m_tree[ $著述碼 ][ $版文檔碼 ], '' );
</pre>

<pre>
Array
(
    [0] => 0003,3,2,1-5_JINGQUAN,0002,2_insert
    [1] => 0003,4,1,1-5_JINGQUAN,0002,3_insert
    [2] => 0003,4,2,1-5_JINGQUAN,0002,5_insert
    [3] => 0003,5,1,1-5_JINGQUAN,0002,7_insert
    [4] => 0003,5,2,1-5_JINGQUAN,0002,9_insert
)
</pre>

在這樣一條路徑中，最重要的是最後這三個部分：去處（基準正文樹的落腳點）、來處（杜著述中文字的位置）、處理方式。當然，`注釋` 與展示單元（presentation component）有關（見下文），得在某地方補回去。最簡單的做法就是把 `注釋` 前置（prepend）到指令之前：

<pre>
注釋_0003,3,2,1-5_JINGQUAN,0002,2_insert
</pre>

或者在執行指令時補回去。

## 後設資料樹的操作

後設資料樹中的路徑，可以通過後設-後設資料來修改。比如說，要把大段的注文分解成幾個較短的注釋，路徑中的落腳點可以分支：

<pre>
0003,3,2,1-5_0003,3,2,1-2_JINGQUAN,0002,2,1_insert
0003,3,2,1-5_0003,3,2,3_JINGQUAN,0002,2,2_insert
</pre>

就是把「《史記·貨殖傳》：泰山之陽則魯，其陰則齊。」分給「齊魯」，「李因篤云：靑，嶽色也。」分給「靑」。利用後設資料樹與後設-後設資料，這樣的樹操作很容易辦到。

同樣地，如果有需要，我們可以在節點 `0003,3,2,1-5` 下再加一個子節點，用以引入《史記·貨殖傳》的原文（可以引入上下文）、白話譯文等。總之，因爲我們用的是樹結構，這類加加減減的樹操作很容易辦到。
