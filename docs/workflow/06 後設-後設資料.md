# 06 後設-後設資料

Status: Draft

---

## 槪述

後設-後設資料指在原始資料、後設資料都完備處理好的基礎上，把原始資料、後設資料之間的聯係，以及對後設資料本身，進行分析、歸類、統計而生成的第三級資料。此類資料對於比較各時代的杜著述、瞭解杜甫研究的發展有很大幫助。

## 杜詩文與杜著述的聯係

- 在一部杜著述中，每段文字（注文、評論等）都有一個主鍵
- 例子：在楊倫《杜詩鏡銓》中，「《舊唐書》：高祖皇后竇氏父毅，於門屛畫二孔雀，有求婚輒與兩箭，潛約中目者許之。高祖後至，兩發各中一目，遂歸於帝。」一段文字的主鍵是 `JINGQUAN,0016,3,7`
- 主鍵是樹路徑，代表文字內容的實際位置
- 此類主鍵又是後設資料樹路徑的組成部分，因此後設資料路徑也是杜著述文字內容的主鍵
- 搜索杜著述的內容，固然可以知道文字內容的位置，同時也可以知道文字內容的語義功能（注釋、評論、引書等）、用在杜詩中甚麼地方
- 反過來，從杜甫的某詩句開始，可以羅列系統內的所有相關的杜著述的內容，幷進行比較
- 由於基準正文樹路徑、杜著述正文樹路徑都容許範圍標記，任何較大的範圍都可以進一步細分
- 基準正文樹中最小的文字單位是字，杜著述中最小的單位是行；只要切分有實際意義，行也可以分行
- 只要二者之間的聯係能滿足需要，這種切分可以進行到最小的單位

## 引用典籍

- `引書`可以是功能分類的一種
- `引書`的範圍（`scope`）仍然是基準正文樹路徑，來源則是某典籍的內容路徑（`src_path`）
- 通過這類後設資料路徑，可以統計某典籍的引用次數、杜詩文位置、在同一或不同的杜著述中的分佈情況等等
- 此類資料可作爲研究杜詩語言的重要參考資料

## 杜著述資料統計

- 在後設資料樹中，像`注釋`、`評論`一類的節點都可以被統計
- 在某杜著述某詩的後設資料樹中，`注釋`的子節點數量，就是這杜著述對這詩作的注文條目的數量
- 把所有後設資料樹中`注釋`子節點數量全加起來，就可以得出全書注文條目的總數
- 把篇目、`注釋`子節點數量攤開來看，可以得到注文條目的分佈情況
- 如果再結合注文的字數、注文中的書名、人名、地名、官職、器物名等，還可以得到其他種類的統計數字
