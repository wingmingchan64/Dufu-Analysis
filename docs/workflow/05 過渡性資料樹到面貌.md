# 05 過渡性資料樹到面貌

Status: Draft

---

## 槪述

過渡性資料樹是展示單元所接受的輸入資料。以過渡性資料樹爲基礎，用者可以按需要生成各種面貌文檔。

---

## 生成 HTML

此系統只負責從文本到過渡性資料樹的生成，幷不負責面貌文檔的生成。原因很簡單，系統輸出的過渡性資料樹可用於各種格式、用途，可能生成的面貌的數量是無限的。

試想象，假如我們有十個杜著述，每兩個杜著述，比如錢謙益《錢注杜詩》、楊倫《杜詩鏡銓》，我們就想生成一個 HTML 混合版本文檔，十個杜著述就可以生成四十五個不同的文檔。如果每三個杜著述算一個組合，就有一百二十個不同組合。如果每四個杜著述算一個組合，就有兩百百一十個不同組合。這還是 HTML 一個格式。
